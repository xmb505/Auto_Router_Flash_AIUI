## 逻辑说明

### 移动/电信版本流程
1. **检测 IP**：循环 ping 多个网段（192.168.2.1, 192.168.10.1, 192.168.31.1），找到在线的 IP
2. **输入密码**：提示用户输入路由器管理密码（明文）
3. **执行刷机**：调用 `test_login1.sh`，传入密码参数
   - 登录 Web UI（SHA1 认证）
   - 获取 stok
   - 开启 SSH（extendwifi_connect + oneclick_get_remote_token）
   - SCP 传输文件到 /tmp/
   - SSH 执行 mtd write bootloader + sysupgrade
4. **成功判断**：脚本返回成功状态
5. **进入第二阶段**：切换到升级模式

### 联通版本流程
1. **检测 IP**：同上
2. **执行刷机**：调用 `unicom_flash.sh`
   - 初始化路由器（设置 WAN 为 DHCP）
   - 计算 ROOT 密码（通过 SN）
   - 开启 SSH
   - 传输文件并刷入
3. **成功判断**：脚本返回成功状态
4. **进入第二阶段**：切换到升级模式

### 失败处理
见「刷机失败」文档

## 小提示
1. Uboot 启动时会短暂占用 192.168.1.1，需要持续 ping 检测
2. 第二阶段为「升级模式」，会刷入最终固件 shareWiFi
3. 成功刷入 bootloader + kernel 后，路由器会重启进入 uboot 模式
4. 联通版本无需手动输入密码，流程完全自动化
