# 新UI设计 - 总体设计方案

## 设计理念

**给工人用的界面，不是给程序员看的**

核心目标：
1. **简单**：不用培训就能用
2. **清晰**：一眼知道当前在做什么
3. **容错**：输错了能纠正，不会刷坏路由器
4. **高效**：减少重复操作，能自动就自动

## 界面组成

```
主菜单
 ├─ CR660X 分支
 │   ├─ 选择运营商
 │   ├─ 密码输入（移动/电信）
 │   ├─ 刷机进行中
 │   ├─ 刷机完成
 │   └─ 刷机失败处理
 │
 └─ JGC 分支
     ├─ 选择步骤（1/2/3）
     ├─ 步骤1进行中
     ├─ 步骤2进行中
     ├─ 步骤3进行中
     └─ 完成统计
```

## 输入规范

### 数字输入
```
请输入数字 [1-3]：_
```
规则：只接受 1-3，其他重新提示

### 密码输入
```
请输入密码：admin888
```
规则：
- 明文显示
- 必须为 8 位，不足8位或超过8位都无效
- 不足8位时提示"密码必须为8位，请重新输入"，清空重新输入
- 超过8位时提示"密码必须为8位，请重新输入"，清空重新输入
- 正好8位：正常通过

### 确认输入
```
确认执行？ [Y/n]：Y
```
规则：默认 Y，输入 n 返回

## 错误提示规范

### 格式
```
┌─────────────────────────────────────────┐
│  ❌ 错误提示                      │
├─────────────────────────────────────────┤
│                                         │
│  具体错误信息...                     │
│                                         │
│  请选择处理方式：                     │
│  [ 1 ] 重试                          │
│  [ 2 ] 返回                          │
│                                         │
└─────────────────────────────────────────┘
```

### 常见错误话术

| 错误类型 | 提示话术 |
|----------|----------|
| 密码错误 | 密码错误，请重新输入 |
| 连接失败 | 无法连接路由器，请检查网线 |
| 文件不存在 | 找不到刷机文件，请联系管理员 |
| 服务未运行 | 下载服务未启动，正在启动... |
| 操作超时 | 操作超时，请重试或联系管理员 |

## 颜色使用规范

### 状态颜色
- ✅ 成功：绿色
- ❌ 失败：红色
- ⚠️ 警告：黄色
- ℹ️ 提示：蓝色
- ⏳ 等待：灰色

### 强调内容
- 重要数字：加粗或高亮
- 当前选项：反色或标记
- 进度条：不同颜色区分状态

## 响应时间规范

| 操作 | 响应时间 | 用户感知 |
|------|---------|---------|
| 输入验证 | <100ms | 即时 |
| 状态刷新 | 1秒 | 适中 |
| 页面切换 | <500ms | 流畅 |
| 刷机进度 | 1-3秒 | 不着急 |
