# 工人版 UI 设计方案

## 设计原则

1. **简洁明了**：减少文字，一眼看懂当前状态
2. **工人友好**：少用技术术语，多用人话
3. **突出重点**：当前状态、下一步做什么一目了然
4. **快速反馈**：操作结果立即显示
5. **错误清晰**：出错了怎么处理直接告诉你

## 界面布局规范

```
┌─────────────────────────────────────────┐
│  ━━━ 路由器刷机工具 v1.0 ━━━       │

├─────────────────────────────────────────┤
│                                         │

│                                         │
├─────────────────────────────────────────┤
│  [选项列表区]                          │
│  [ 1 ] 选项说明                        │
│  [ 2 ] 选项说明                        │
├─────────────────────────────────────────┤
│  [状态栏]  已完成: 12 台  在线: IP地址  │
└─────────────────────────────────────────┘
```

## 颜色方案（终端）

- **绿色** ✅：成功、完成、在线、运行中
- **红色** ❌：失败、错误、离线、停止
- **黄色** ⚠️：警告、等待中、部分完成
- **蓝色** ℹ️：提示、说明
- **白色/灰色**：普通文字

## 输入规范

- **数字选择**：`[ 1 ]` 等待用户输入 1-9
- **密码输入**：明文显示，带星号标明：`密码：admin888 [输入完按回车]`
- **确认操作**：`[Y/n]` 默认 Y

## 流程简化原则

### 当前设计的问题
- ❌ 技术术语太多："官放系统破解刷入BOOTLOADER和KERNEL"
- ❌ 说明太长："可以在本地创建一个mac地址，流程是先ping到刷机完成状态IP..."
- ❌ 状态不明确：不知道当前在做什么

### 优化后
- ✅ 用人话："步骤 1：更换路由器系统"
- ✅ 简化说明："自动完成，请等待"
- ✅ 状态清晰："正在刷机 [████████░░] 80%"
