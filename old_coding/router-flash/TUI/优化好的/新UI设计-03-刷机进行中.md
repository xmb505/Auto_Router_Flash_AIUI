# 新UI设计 - 刷机进行中（通用状态界面）

## 界面展示

```
┌─────────────────────────────────────────┐
│  CR660X 路由器刷机                 │
│  步骤 1/3：刷入引导系统             │
├─────────────────────────────────────────┤
│                                         │
│  当前状态：正在刷机 ...               │
│  进度： [████████████░░░░░░░] 70%   │
│  预计剩余：约 30 秒                  │
│                                         │
│  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━   │
│                                         │
│  ⏳ 请耐心等待，不要关闭窗口         │
│                                         │
├─────────────────────────────────────────┤
│  路由器 IP：192.168.10.1             │
│  按键 [Q] 取消刷机                     │
└─────────────────────────────────────────┘
```

## 优化点

1. **进度条可视化**：用字符画进度条，直观显示进度
2. **步骤标识**：清楚当前是第几步/共几步
3. **状态简洁**：用短句代替长描述
4. **操作提示**：告诉工人现在该做什么（等待）
5. **可取消**：提供取消选项，防止错误刷机

## 状态文字规范

| 阶段 | 显示文字 |
|------|---------|
| 检测中 | 🔍 正在检测路由器... |
| 连接中 | 📡 正在连接路由器... |
| 传输中 | 📤 正在传输文件... [50%] |
| 刷写中 | ⚙️ 正在刷写系统... [70%] |
| 重启中 | 🔄 路由器正在重启... |
| 完成 | ✅ 刷机完成！ |

## 进度条格式

```python
# 伪代码
def render_progress(percent):
    bar_length = 30
    filled = int(bar_length * percent / 100)
    bar = "█" * filled + "░" * (bar_length - filled)
    return f"[{bar}] {percent}%"
```

## 交互逻辑

```
刷机进行中 →
  ├─ 每秒刷新进度
  ├─ 按 [Q] 可以取消
  ├─ 完成后进入下一步
  └─ 失败后进入失败界面
```

## 工人操作指南

1. **看到这个界面**：说明刷机正常进行，请等待
2. **进度不动了**：等待 1 分钟，还是不动说明卡住了
3. **想停止**：按 Q 键取消，但刷机一半停止可能损坏路由器
4. **意外中断**：路由器断网、断电等，界面会报错
