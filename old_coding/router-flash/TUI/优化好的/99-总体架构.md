## 核心功能模块

### 1. 路由器检测模块（Router Detector）
**功能**：ping 多个网段，找到在线的路由器 IP

**检测的 IP 列表**：
- 192.168.1.1（uboot）
- 192.168.2.1（JGC-Qx 官方）
- 192.168.10.1（JGC-Q10 官方/CR660X 官方）
- 192.168.31.1（CR660X 小米官方）
- 192.168.123.1（PDCN 系统）
- 10.11.12.1（最终固件）

**实现**：
```python
def detect_router_ip(ip_list):
    """遍历 IP 列表，返回所有在线的 IP"""
    online_ips = []
    for ip in ip_list:
        if ping(ip, count=1, timeout=1) == 0:
            online_ips.append(ip)
    return online_ips
```

### 2. MAC 地址获取模块（MAC Getter）
**功能**：ping 后获取目标 IP 的 MAC 地址

**实现**：

### 4. HTTP 服务器管理模块（HTTP Server Manager）
**功能**：自动启动和管理 chfs 文件服务器

**实现**：
```python
def start_chfs_if_needed():
    """检查并启动 chfs 服务器"""
    if is_chfs_running():
        return True

    # 后台启动 chfs
    cmd = f"cd /home/xmb505/alpine-router-flash/JGC-Qx && nohup ./chfs-linux-amd64-3.1 &"
    os.system(cmd)

    # 等待启动
    time.sleep(2)

    return is_chfs_running()

def is_chfs_running():
    """检查 chfs 是否运行"""
    try:
        result = os.popen("pgrep -f chfs-linux-amd64").read()
        return result.strip() != ""
    except:
        return False

def get_chfs_status():
    """获取 chfs 状态字符串"""
    return "RUNNING" if is_chfs_running() else "STOPPED"
```

### 5. 脚本调用模块（Script Executor）
**功能**：调用各个刷机脚本并处理输出

**CR660X 脚本**：
- `test_login1.sh`（移动/电信版本，需要密码）
- `unicom_flash.sh`（联通版本，自动）

**JGC 脚本**：
- `official_flash_all.py`（步骤 1）
- `pdcn_put_bootloader.py`（步骤 2.1）
- `pdcn_flash_bootloader.py`（步骤 2.2）
- `pdcn_reboot.py`（步骤 2.3）
- `uboot_flash.py`（步骤 3）

**实现**：
```python
def run_script(script_path, args=None, timeout=300):
    """执行脚本并返回结果"""
    cmd = [script_path]
    if args:
        cmd.extend(args)

    result = subprocess.run(cmd, capture_output=True, text=True, timeout=timeout)

    return {
        "returncode": result.returncode,
        "stdout": result.stdout,
        "stderr": result.stderr
    }
```

## 状态机（State Machine）

```
START → SELECT_MODEL → [CR660X | JGC]

[CR660X 分支]:
SELECT_MODEL → SELECT_CARRIER → [MOBILE | UNICOM]
    MOBILE → INPUT_PASSWORD → EXECUTE_FLASH → [SUCCESS | FAIL]
    UNICOM → EXECUTE_FLASH → [SUCCESS | FAIL]
    SUCCESS → WAIT_UBOOT → EXECUTE_UPGRADE → COUNT_INVENTORY → RETURN_MAIN
    FAIL → [VIEW_LOG | RETRY | RETURN_MAIN]

[JGC 分支]:
SELECT_MODEL → SELECT_FLOW → [STEP1 | STEP2 | STEP3]
    STEP1 → DETECT_IP → [AUTO_PASSWORD | MANUAL_PASSWORD] → EXECUTE_STEP1 → [SUCCESS | FAIL]
    STEP2 → DETECT_IP → EXECUTE_STEP2 → [SUCCESS | FAIL]
    STEP3 → DETECT_IP → EXECUTE_STEP3 → [SUCCESS | FAIL]
    SUCCESS (STEP1) → STEP2
    SUCCESS (STEP2) → STEP3
    SUCCESS (STEP3) → COUNT_INVENTORY → RETURN_MAIN
    FAIL → [RETRY | RETURN_MAIN]
```

## 错误处理策略

### 1. 网络错误
- **ping 失败**：提示检查网络连接，返回主菜单
- **HTTP 请求失败**：提示检查路由器状态，重试 3 次
- **SSH/SCP 失败**：提示检查密码或 SSH 连接，显示错误日志

### 2. 脚本执行错误
- **超时**：5 分钟无响应视为超时，提示查看日志
- **返回非零**：显示失败信息，提供重试或查看日志选项
- **进程崩溃**：显示错误信息，记录到日志文件

### 3. 文件错误
- **固件文件不存在**：提示检查文件路径，停止刷机
- **库存文件损坏**：提示重置库存记录或修复 JSON

## 实现建议

### 使用库
- **Rich**：用于美化终端 UI（进度条、颜色、表格）
- **Click**：用于命令行参数解析（如果有 CLI 模式）
- **APScheduler**：用于定时任务（如定期检测路由器上线）

### 并发处理
- 所有刷机操作为单线程（避免网络冲突）
- HTTP 服务器后台运行（subprocess 或 threading）
- ping 检测可以使用多线程加速（同时 ping 多个 IP）

### 输入验证
- 数字输入：只允许 1-9
- 密码输入：不遮掩（用户明确要求）
- IP 输入：验证格式（可选）

### 界面刷新
- 使用 `curses` 或 Rich 的 Live Display
- 定期更新状态（每秒刷新一次）
- 等待用户输入时暂停刷新
