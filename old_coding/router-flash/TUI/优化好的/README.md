## 快速导航

### CR660X 路由器刷机流程
1. 选择运营商（移动/电信需要输入密码，联通自动）
2. 刷入 bootloader + kernel
3. 等待 uboot 上线
4. 刷入最终固件
5. 检测 10.11.12.1 上线，统计库存

### JGC 路由器刷机流程
1. 官方固件 → PDCN 系统（自动或手动密码）
2. PDCN 系统 → bootloader（需要 HTTP 服务器）
3. bootloader → 最终固件
4. 检测 10.11.12.1 上线，统计库存

## 核心模块

- **路由器检测**：ping 多个网段，找到在线 IP
- **MAC 获取**：`ip neigh show` 获取设备 MAC 地址
- **库存管理**：记录已刷机设备，统计数量
- **HTTP 服务器**：自动启动 chfs 文件服务器
- **脚本执行**：调用各阶段刷机脚本
- **界面渲染**：根据状态动态显示界面

## 注意事项

1. **密码输入不遮掩**：用户明确要求明文显示，方便确认
2. **uboot 检测优化**：uboot ping 行为不稳定，需增加 HTTP 探测
3. **HTTP 服务器依赖**：JGC 步骤 2 必须运行 chfs 服务器
4. **库存记录**：CR660X 和 JGC 共用 `mac_inventory.json`
5. **最终系统 IP**：两种路由器最终系统 IP 均为 10.11.12.1

## 实现建议

- 使用 **Rich** 库美化终端 UI
- 使用 **curses** 实现动态刷新界面
- 所有刷机操作单线程执行，避免网络冲突
- 日志记录到文件，便于排查问题
