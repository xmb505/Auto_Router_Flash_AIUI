## 逻辑说明

### 升级模式流程
1. **等待 Uboot 上线**：持续 ping 192.168.1.1，直到路由器进入 uboot 模式
2. **内存检查**：SSH 连接 uboot（密码为空），检查内存是否 >128M
   - 代码中已有 128M 检测逻辑，但写得不完善，需要注意
3. **文件传输**：SCP 传输 shareWiFi 固件到 /tmp/
4. **执行升级**：SSH 执行 `sysupgrade /tmp/sharewifi_1.0.X.bin`
5. **等待重启**：等待路由器重启并进入目标系统
6. **检测上线**：持续 ping 目标 IP（10.11.12.1），确认刷入成功
7. **库存统计**：
   - 通过 MAC 地址记录已刷机设备
   - 本地存储文件：`mac_inventory.json`
   - 检测到新 MAC 地址 → 计件数 +1，并写入记录文件
8. **返回主菜单**：2 秒后返回 CR660X 主菜单，可刷下一台

### 库存记录机制
```python
# 伪代码
current_ip = "10.11.12.1"
mac = get_mac_from_ip(current_ip)
inventory = load_inventory("mac_inventory.json")

if mac not in inventory:
    inventory[mac] = {
        "ip": current_ip,
        "timestamp": now(),
        "model": "CR660X"
    }
    save_inventory(inventory)
    count = len(inventory)
    print(f"库存无记录，计件数：{count}")
else:
    count = len(inventory)
    print(f"已刷过此设备，计件数：{count}")
```

1. 内存检测代码不完善，需要优化（参考 `update.sh` 中现有逻辑）
2. 目标系统 IP 为 10.11.12.1，与其他阶段不同
