## 逻辑说明

### IP 检测逻辑
循环 ping 多个网段，显示所有在线的 IP：
- 192.168.2.1（JGC-Qx 官方固件）
- 192.168.10.1（JGC-Q10 官方固件）
- 192.168.123.1（PDCN 系统）
- 192.168.1.1（Uboot）
- 10.11.12.1（最终系统）

如果没检测到任何 IP，显示"这些没检测到IP"

### 流程选择
**输入 1：官方系统全自动刷入 PDCN**
- 自动调用 `official_flash_all.py`
- 完成后自动进入步骤 2（PDCN 刷 bootloader）

**输入 2：PDCN 刷入 BOOTLOADER**
- 等待 192.168.123.1 上线
- 自动调用 `pdcn_flash_all.py`
- 需要启动 chfs 文件服务器

**输入 3：BOOTLOADER 刷入指定镜像**
- 等待 192.168.1.1 上线
- 自动调用 `uboot_flash.py`
- 不需要 chfs 文件服务器

### HTTP 服务器自动启动
- 进入此页面时，自动检查并启动 `chfs-linux-amd64-3.1`
- 进程管理：使用 `pgrep` 或 `ps aux` 检测是否运行
- 如果未运行，后台启动：`nohup ./chfs-linux-amd64-3.1 &`
- 端口绑定：192.168.123.2:8080
- 状态显示：RUNNING（绿色）/ STOPPED（红色）

### 库存统计
- 与 CR660X 使用相同的库存记录机制
- 文件：`mac_inventory.json`
- 检测到最终系统（10.11.12.1）上线后更新计数

## 小提示
1. HTTP 服务器必须在 PDCN 刷 bootloader 阶段运行，否则路由器无法 wget 下载文件
2. 流程 1 → 2 → 3 顺序执行，也可以从中间某个阶段开始
3. 如果路由器已经在 PDCN 系统，可以直接选择流程 2
4. 如果路由器已经在 bootloader（能 ping 到 192.168.1.1），可以直接选择流程 3
5. 所有阶段完成后，最终系统 IP 为 10.11.12.1
