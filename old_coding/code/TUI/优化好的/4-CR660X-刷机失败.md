# CR660X 刷机失败处理

## 界面展示
```
-----CR660X/TR660X 批量刷机模式-----
所处模式：（1：刷机模式）
选择运营商：（1：中国移动/中国电信）
检测到IP：（检测到的IP）
请输入路由器管理密码：
正在执行系统破解脚本。。
FAILED！刷入失败！
输入1查看失败日志，输入2重试，输入3返回主菜单：（用户输入数字）


-----Powered by Haku Studio 2026-----
```

## 逻辑说明

### 失败判断条件
- `test_login1.sh` 或 `unicom_flash.sh` 返回非零退出码
- 超时（脚本执行超过 3 分钟未返回）
- 网络连接失败（SSH/SCP/HTTP 请求失败）

### 用户选择处理
**输入 1：查看失败日志**
- 显示脚本输出的完整日志
- 包括错误信息、ping 结果、HTTP 响应等
- 查看完后返回失败界面，重新选择

**输入 2：重试**
- 重新执行当前流程
- 保留之前输入的密码（移动/电信版本）
- 如果是密码错误，可以重新输入密码

**输入 3：返回主菜单**
- 返回 CR660X 主菜单（开始界面）
- 可以选择不同运营商或返回主界面

### 联通版本失败
界面展示同上，但无密码输入步骤：
```
-----CR660X/TR660X 批量刷机模式-----
所处模式：（1：刷机模式）
选择运营商：（2：中国联通）
检测到IP：（检测到的IP）
正在执行系统破解脚本。。
FAILED！刷入失败！
输入1查看失败日志，输入2重试，输入3返回主菜单：（用户输入数字）


-----Powered by Haku Studio 2026-----

联通版本（恢复出厂后不要密码，第二阶段不用管）
```

## 小提示
1. **密码输入不要遮住**（用户明确要求），方便用户确认是否输错
2. 常见失败原因：
   - 密码错误
   - 路由器未在官方固件状态（已经刷过）
   - 网络连接问题
   - 路由器 MAC 地址不在 ARP 表中（检查 `ip neigh show`）
3. 查看日志可以帮助定位问题，建议将日志保存到文件以便排查
4. 重试时会重新检测 IP（因为路由器可能重启后 IP 变化）
5. 联通版本失败也可能是因为未恢复出厂设置
