# 新UI设计 - 密码输入界面

## 界面展示

```
┌─────────────────────────────────────────┐
│  CR660X 路由器刷机                 │
│  步骤 1/2：刷入引导系统             │
├─────────────────────────────────────────┤
│                                         │
│  已检测到路由器：192.168.10.1       │
│                                         │
│  请输入路由器管理密码：               │
│                                         │
│  ┌─────────────────────────────────┐   │
│  │ admin888                      │   │
│  └─────────────────────────────────┘   │
│                                         │
│  [输入完按回车]                        │
│                                         │
│  💡 提示：密码在路由器背面贴纸上 │
│                                         │
├─────────────────────────────────────────┤
│  按键 [Esc] 返回                      │
└─────────────────────────────────────────┘
```

## 优化点

1. **密码框效果**：用字符画一个输入框，更直观
2. **提示位置**：底部小字提示，不打扰输入
3. **返回选项**：想返回按 Esc
4. **输入状态明确**：`[输入完按回车]` 告诉工人怎么确认

## 输入框渲染

```python
# 伪代码
def render_input_box(value="", width=30):
    top = "┌" + "─" * (width - 2) + "┐"
    mid = "│ " + value + " " * (width - len(value) - 3) + "│"
    bottom = "└" + "─" * (width - 2) + "┘"
    return "\n".join([top, mid, bottom])
```

## 交互逻辑

```
密码输入 →
  ├─ 显示输入框
  ├─ 等待用户输入（明文显示）
  ├─ 按回车 → 验证密码
  │   ├─ 正确 → 开始刷机
  │   └─ 错误 → 清空输入框，重新输入
  └─ 按 Esc → 返回上一界面
```

## 提示信息规范

| 场景 | 提示信息 |
|------|---------|
| 首次输入 | 💡 提示：密码在路由器背面贴纸上 |
| 密码错误 | ❌ 密码错误，请重新输入 |
| 网络错误 | ⚠️ 无法连接路由器，请检查网线 |
| 已尝试3次 | ⚠️ 已尝试 3 次，按 Esc 返回 |

## 工人操作指南

1. **密码在哪里**：看路由器背面或底部，有管理密码
2. **修改过密码**：用你自己设置的密码
3. **输错了**：直接继续输，会覆盖之前的
4. **不知道密码**：按 Esc 返回，恢复出厂设置
5. **明文显示**：密码会显示出来，请确认输对
