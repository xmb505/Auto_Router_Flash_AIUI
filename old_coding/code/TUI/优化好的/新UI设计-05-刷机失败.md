# 新UI设计 - 刷机失败处理

## 界面展示

```
┌─────────────────────────────────────────┐
│  ❌ 刷机失败！                      │
├─────────────────────────────────────────┤
│                                         │
│  失败原因：密码错误                  │
│                                         │
│  可能的原因：                           │
│  1. 路由器密码输入错误              │
│  2. 路由器不在官放固件状态        │
│  3. 网络连接问题                    │
│                                         │
│  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━   │
│                                         │
│  请选择操作：                           │
│                                         │
│  [ 1 ] 查看详细日志                  │
│  [ 2 ] 重新输入密码重试             │
│  [ 3 ] 返回主菜单                    │
│                                         │
└─────────────────────────────────────────┘

请输入数字 [1-3]：
```

## 优化点

1. **失败原因放前面**：工人第一眼看到是为什么失败
2. **可能原因列表**：用数字列出，清晰明了
3. **提供解决方案**：每个失败原因都有对应操作
4. **日志查看**：可以看详细日志方便排查
5. **返回选项**：失败后可以返回主菜单

## 交互逻辑

```
失败处理 →
  ├─ 显示失败原因 + 可能原因列表
  ├─ [1] 查看日志 → 显示日志，查看完回到此界面
  ├─ [2] 重试 → 回到密码输入界面
  └─ [3] 返回 → 回到主菜单
```

## 失败原因模板

| 场景 | 显示原因 | 可能原因 |
|------|---------|---------|
| 密码错误 | 密码错误 | 路由器密码输入错误 |
| 连接失败 | 连接路由器失败 | 网络断开、路由器不在线 |
| 超时 | 操作超时 | 路由器卡住、网络慢 |
| 文件错误 | 固件文件错误 | 固件损坏、路径不对 |

## 工人操作指南

1. **密码错误**：检查路由器背面贴纸，重新输入
2. **连接失败**：检查网线，确认路由器亮绿灯
3. **想换路由器**：按 3 返回主菜单，换一台再刷
4. **查看日志**：按 1 看详细错误信息，找技术人员帮忙
