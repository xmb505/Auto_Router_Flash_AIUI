# AX3600 刷机全玩法

刷机玩法三部曲：初始化 → 开 SSH → 刷 LibWrt。
可逆操作：随时切回 stock 或切到 LibWrt。

## 全链路速查

```bash
cd src/project/ax3600
# 默认密码: 12345678, SSH: root/root, IP: 192.168.31.1 (stock) / 192.168.1.1 (OpenWrt)
```

### 一、1.0.x 工厂态初始化 + 开 SSH

1.0.x 出厂固件直接支持注入开 SSH（1.1.x 已被封堵）。

```bash
# Step 1: 出厂初始化 (设管理密码 + Wi-Fi)
python3 1.official_init.py --admin-pwd 12345678

# Step 2: 拿 stok
python3 2.login_get_stok.py --pwd 12345678

# Step 3: 注入开 SSH (set_config_iotdev ssid 漏洞)
python3 3.enable_ssh.py --wait
# SSH: root@192.168.31.1 / 密码: root
```

### 二、1.1.x 工厂态迂回（降级 → 初始化 → SSH）

1.1.x 上 `set_config_iotdev` 注入已被封堵，需要先降级到 1.0.17。

```bash
# Step 1: 出厂初始化
python3 1.official_init.py --admin-pwd 12345678

# Step 4: 降级到 1.0.17 (recovery=1 清 NVRAM)
python3 4.official_upgrade.py --file files/R3600_1.0.17.bin
# 等 ~45s 重启

# 重新初始化 (1.0.17 工厂态)
python3 1.official_init.py --admin-pwd 12345678
python3 2.login_get_stok.py --pwd 12345678
python3 3.enable_ssh.py --wait
```

### 三、刷 LibWrt（SSH + ubiformat 到非活跃分区）

有 SSH 后，把 LibWrt 刷到**对侧 mtd**，双系统互不干扰。

```bash
# 上传 LibWrt .ubi 到 /tmp
./5.firmware_upload_on_miwifi.sh \
  --file files/libwrt-25.12.0-qualcommax-ipq807x-xiaomi_ax3600-stock-squashfs-factory.ubi \
  --ip 192.168.31.1

# 自动探测当前 mtd, 烧到对侧 (--part 可选, 不传自动选)
python3 6.miwifi_2_openwrt.py --file-name libwrt.ubi

# 切启动分区到新 mtd + reboot
./set_miwifi_uboot_partition.sh --part 1
./miwifi_ssh.sh --cmd reboot
```

重启后:
- IP: `192.168.1.1`
- SSH: `root@192.168.1.1` (免密, ED25519)
- LuCI: `http://192.168.1.1/cgi-bin/luci`

### 四、切回另一侧分区

当前运行的系统不一定是原厂——可能两个都是 OpenWrt，也可能两侧都刷过。
切分区只改启动 flag，不做内容假设。

```bash
./switch_to_stock.sh
# 等 ~45s 重启 → 192.168.31.1 (如果 mtd12 是 stock)
```

等价手动操作：
```bash
ssh root@192.168.1.1 \
  "fw_setenv flag_try_sys1_failed 0 && \
   fw_setenv flag_try_sys2_failed 1 && \
   fw_setenv flag_boot_rootfs 0 && \
   reboot"
```

### 五、应用自定义 overlay（主题/配置等）

LibWrt 跑起来后，把个性化配置铺到 `/overlay/upper/`：

```bash
# 打包 overlay (顶层必须 overlay/ 目录)
tar -czf overlay-new.tar.gz -C /path/to/files overlay

# 上传 + 解压到 /overlay/upper/ + 重启
python3 7.custom_openwrt.py --file overlay-new.tar.gz
```

详见 [`custom-openwrt.md`](custom-openwrt.md)。

### 六、双系统互切

```
         mtd12 (rootfs)        mtd13 (rootfs_1)
       ┌────────────────┐  ┌────────────────────┐
       │   分区 A        │  │    分区 B          │
       │ 192.168.31.1?   │  │ 192.168.1.1?       │
       └───────┬──────────┘  └────────┬────────────┘
               │                       │
               │ ┌─────────────────┐  │
               │ │ set_miwifi_uboot│  │
               │ │ _partition.sh   │  │
               └─│     --part 1    ├──┘
               ┌─│     --part 0    ├──┐
               │ └─────────────────┘  │
               │                       │
               └───────┬──────────────┘
                       │
            物理 reset / 恢复出厂
```

- `set_miwifi_uboot_partition.sh --part 0` → 切到 mtd12 (rootfs)
- `set_miwifi_uboot_partition.sh --part 1` → 切到 mtd13 (rootfs_1)
- `switch_to_stock.sh` → 从当前系统切到对侧（假设对侧有启动系统）

## boot flag 原理

AX3600 用 3 个互补 nvram flag 控制下次启动哪个分区：

| `flag_try_sys1_failed` | `flag_try_sys2_failed` | `flag_boot_rootfs` | 下次启动 |
|------------------------|------------------------|---------------------|----------|
| 0                      | 1                      | 0                   | mtd12 (rootfs) |
| 1                      | 0                      | 1                   | mtd13 (rootfs_1) |

**三 flag 必须一起设**，跟 AX6 一样。只有 `flag_boot_rootfs` 不行——uboot 同时检查 `flag_try_sysN_failed` 来最终决策。

**验证极性**: `flag_boot_rootfs=0` → mtd12, `=1` → mtd13。
（早期实验中被 kernel 启动后把 flag 复位 0 误导了极性判断，最终以 `/proc/cmdline` 为准。）

## 注入开 SSH 原理

```python
ssid = "-h; <命令> ;"
POST /api/misystem/set_config_iotdev?bssid=Xiaomi&user_id=longdike&ssid=<ssid>
```

`ssid` 参数中的 `-h` 触发 hostapd 帮助模式，`;` 是 shell 命令分隔符。

**当前验证过的固件**：
- 1.0.17 ✅ 4 步注入全部成功（nvram + sed + passwd + dropbear restart）
- 1.1.25 ❌ 连合法 SSID 都返 `code:1523 参数错误`

## SSH 连接速查

| 场景 | 命令 |
|------|------|
| LibWrt SSH | `ssh root@192.168.1.1` 密码 admin |
| stock SSH | `ssh -oHostKeyAlgorithms=+ssh-rsa root@192.168.31.1` 密码 root |
